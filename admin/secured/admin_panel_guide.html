1. Access System

The system uses an email whitelist for admin access. The entry page verifies the logged-in email ‚Äî if it is not on the list the user is redirected back to the verification entry page (no message). Every page load re-runs this verification check to maintain integrity.

Rules

  • Login ‚Üí email checked against admin whitelist ‚Üí Dashboard if allowed.
  • If not allowed ‚Üí redirect to entry verification page automatically.
  • Continuous validation on each module load.

3. USER DATA

Purpose: Read-only monitoring of user accounts for GDPR and operations.

Fields shown

  • Join Date
  • Last Login
  • Public Posts
  • Slug (handle)
  • Terms Accepted (boolean)
  • Verified (admin flag)
  • Terms Timestamp

Profile images

Only the one-time custom profile image uploaded by the user (stored in Supabase) is shown. Google profile images are not surfaced here.

Actions

Admins cannot edit or delete user data in-panel. For bans, restrictions, or account edits email IT with the user slug/UID and justification.

Displayed

5. CARDS MANAGER

Create lightweight content cards used across the site. Creation is permitted; editing existing cards is restricted.

Fields & limits

  • Title ‚Äî required, max 300 characters
  • Description ‚Äî optional, max 10,000 characters
  • Image URL ‚Äî must be HTTPS from allowed CDN (Dropbox / Supabase)

Image flow

Upload images via Portfolio ‚Üí Item Manager. Grab the CDN URL and paste it here.

Cards are view-only after creation. To remove or edit a card, email IT with details.

6. PORTFOLIO

All visual assets are managed here. Two submodules: Item Manager (upload pipeline) and Folder Manager (organisation).

6.1 Item Manager

Purpose: Upload images to Dropbox CDN and register URLs in Firestore.

Capabilities

  • Upload up to 5 images per batch
  • Original preview, file size, aggregated storage calculation
  • CDN URL returned after upload
  • Pagination and a frontend console for logs

Flow

Frontend -> Cloudflare Worker -> Dropbox -> Worker returns CDN URL -> Save URL to Firestore

Notes

  • Show network warning for slow connections
  • Allowed formats: JPG, PNG, WEBP (max size set by IT)
  • Buttons: Select up to 5 images, Upload to CDN, Copy URL, View original

Storage quotas

  • Supabase (profile images) ‚Äî 1 GB (private)
  • Dropbox (portfolio media) ‚Äî 2 GB (serves portfolio)

6.2 Folder Manager

Purpose: Organize images into folders for the portfolio page.

Top controls

  1. Create Folder ‚Äî title, description, order, visibility
  2. View Unassigned Images
  3. Fix Missing Fields ‚Äî auto-fill metadata for recent uploads (run once after upload)

Folder fields

  • Title ‚Äî required
  • Description ‚Äî optional
  • Order ‚Äî integer (lower = shown earlier)
  • Visibility ‚Äî Public / Hidden

Assignment

Assign by checkbox or drag/drop UI; console logs track assignments and fixes. "Fix Missing Fields" auto-populates basic metadata‚Äîuse sparingly as recommended by IT.
