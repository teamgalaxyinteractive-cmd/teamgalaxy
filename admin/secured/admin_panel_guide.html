Overview

GalaxyWeb Admin Panel is a secure, access-controlled interface for managing site assets, content, and agreements. Only verified admin emails may access the dashboard. This guide outlines behavior, constraints, and operational flows for every module in the admin panel.

What’s included in this document

  • Access rules & verification flow
  • Dashboard navigation & Contact IT
  • User Data (read-only)
  • Repo Status (read-only file browser)
  • Cards Manager (create cards)
  • Portfolio (Item Manager & Folder Manager)
  • Licenses & Agreement Access modules

1. Access System

This system uses an email-based whitelist for admin access. The entry page verifies if the logged-in email belongs to the admin list. If not, the user is silently redirected back to the verification entry page. There is no "access denied" message shown — the UX returns them to the starting verification gate.

Verification rules

  • On login attempt the system checks the user email against admin whitelist.
  • If allowed, the Dashboard opens.
  • If not allowed, the user is redirected back to entry verification page.
  • Every page load and module navigation re-runs this verification check (continuous validation).

Implementation notes (for devs)

/* PSEUDO onPageLoad() { if (!isAdmin(currentUser.email)) { window.location.href = '/entry-verification'; } } */

Note: Sessions are not forcibly short-lived. The system uses normal session behavior (no short TTL required by design).

2. Dashboard

The Dashboard is the navigation hub. Each module is presented as a tile with a short description and quick metadata (last-checked timestamp, usage hints).

Available tiles / modules

  • USER DATA
  • REPO STATUS
  • CARDS MANAGER
  • PORTFOLIO (Item Manager, Folder Manager)
  • LICENSES
  • AGREEMENT ACCESS

Contact IT (single central action)

Instead of adding per-page report buttons, the Dashboard contains a single Contact IT button which composes an email to squadvertex24official@gmail.com.

3. USER DATA

Purpose: Read-only monitoring of user data for GDPR and operational oversight.

Displayed fields

  • Join Date
  • Last Login
  • Public Posts
  • Slug (handles as public identifier)
  • Terms Accepted (boolean)
  • Verified (admin-only flag)
  • Terms Timestamp

Profile images

Only the custom image uploaded by the user via in-app profile is shown (from Supabase). Google account images are not surfaced here. Users are allowed only one profile photo change to limit storage abuse and to reduce potential misuse (e.g., NSFW uploads).

Permissions & Actions

  • Admins cannot edit or delete user records in-panel.
  • For bans, restrictions, or edits admins must email IT with user slug/uid & justification.
  • All edit requests are processed by IT (squadvertex24official@gmail.com).

4. REPO STATUS

Purpose: Read-only browser for the GitHub repository that powers the site.

What it shows

  • Repository folder tree
  • File contents preview (read-only)
  • File sizes & structure

What it does NOT show

  • No commit history / no commit metadata (by design)
  • No edit capability

Important constraints

  • This module uses API tokens and can be token-costly — recommended for IT use only
  • Admins should limit usage to occasional monitoring only to conserve tokens
  • Use the "Request IT" flow via Dashboard Contact IT if change is required
INFRAS: - Hosting: GitHub - Database: Firebase - Portfolio CDN: Dropbox (2 GB quota) - Profile images: Supabase (1 GB quota)

Note: The repo visible here is the authoritative source for website code and server configuration. Admins should not attempt code changes here.

5. CARDS MANAGER

Purpose: Create blog / announcement cards used across GalaxyWeb. Cards are lightweight content pieces — create allowed, edits restricted.

Create Card — Fields & Validation

  • Title — required. Maximum 300 characters.
  • Description — optional. Maximum 10,000 characters.
  • Image URL — Must be an HTTPS URL from the authorised CDN (Dropbox or Supabase).

Process

  1. Upload images through Portfolio → Item Manager to get CDN url(s).
  2. Open Cards Manager, create new card and paste image URL.
  3. Cards list remains view-only after creation; editing/removal requires contacting IT.
Why limits? Large descriptions are supported but we place conservative caps to prevent accidental content bloat; feel free to ask IT before adding extremely long content.

6. PORTFOLIO

The Portfolio module manages all visual assets for the public portfolio page. It contains two submodules: Item Manager (upload pipeline) and Folder Manager (organization).

6.1 Item Manager

Purpose: Upload images to the Dropbox CDN and register resulting URLs into Firestore.

Capabilities

  • Upload up to 5 images per batch to limit server load
  • Show original preview and file size for each image
  • Calculate and display aggregated storage usage for portfolio space
  • Provide CDN URL after successful upload
  • Pagination for listings
  • Integrated frontend console panel that logs each step

Upload flow (technical)

Frontend -> Cloudflare Worker -> Dropbox upload -> Worker receives Dropbox response & returns CDN URL -> Save CDN URL into Firestore

UI / UX Notes

  • Show network warning if connection is slow or intermittent
  • File limits & allowed formats displayed (e.g., JPG, PNG, WEBP). Max file size enforced by IT (visible in UI)
  • After upload, show "Copy URL" and "View Original" actions
  • Console shows: queued -> uploading -> worker call -> dropbox response -> firestore write

Storage quotas

  • Supabase (profile images): 1 GB — private
  • Dropbox (portfolio media): 2 GB — serves portfolio assets

6.2 Folder Manager

Purpose: Organize uploaded images into folders used by public portfolio page.

Top controls

  1. Create Folder — Provide Title, Description, Order, Visibility
  2. View Unassigned Images — See images not assigned to any folder
  3. Fix Missing Fields — Automated metadata fixer for recent uploads (recommended to run once after upload)

Folder fields

  • Title — required
  • Description — optional
  • Order — integer (lower appears earlier)
  • Visibility — Public or Hidden

Assignment & interface

  • Assign images by checkbox or drag/drop (UI choice)
  • Unassigned images are always visible in the "available" pane
  • Console panel shows assign/unassign events and metadata fixes
"Fix Missing Fields" will try to populate empty metadata fields (category, alt text). IT recommends minimal use — use once after batch uploads.
